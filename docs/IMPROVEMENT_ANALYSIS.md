## 👩‍🎓 실제 사용자 관점 (중고등학생)

### ❌ 현재 불만 사항 (리서치 결과)

| 문제점 | Chalk 현황 | 심각도 |
|--------|------------|--------|
| **오답노트 만들기 번거로움** | ⚠️ 사진 업로드 필요 | 🔴 높음 |
| **복습 주기 자동 설정 없음** | ❌ 미구현 | 🔴 높음 |

| **수식 입력 불편** | ❌ 미구현 | 🟡 중간 |

### 💡 사용자들이 원하는 것

1. **"자동으로 오답 정리해줘"** - 사진만 찍으면 끝
2. **"복습 알림 보내줘"** - 망각 곡선 고려
3. **"비슷한 문제 풀어보게"** - 실시간 추천
4. **"내가 얼마나 늘었는지"** - 성장 시각화

### 🎯 Chalk 개선 우선순위

```
1순위: 복습 알림 시스템 (푸시)
2순위: 유사 문제 실시간 생성

```

## 🧠 ML 엔지니어 관점

### ⚠️ 현재 위험 요소

| 문제 | 설명 | 대응 필요 |
|------|------|----------|
| **Data Drift** | 실제 학생 입력 ≠ 학습 데이터 | 🔴 |
| **Training-Serving Skew** | 개발 환경 ≠ 운영 환경 | 🔴 |
| **Offline vs Online Gap** | 99% 테스트 ≠ 실제 성능 | 🔴 |
| **Zombie Model** | 배포 후 모니터링 안 함 | 🟡 |

### 📊 99% 정확도의 함정

```
현재 상황:
- 학습 데이터: 2,171개 (합성 데이터)
- 테스트 데이터: 326개 (같은 분포)
- 정확도: 99.39%

⚠️ 문제:
- 실제 학생 입력은 다를 수 있음
  예) "3번 틀림", "계산 망함", "몰라ㅜㅜ"
- 이런 입력에 대한 테스트 없음!
```

### 💡 ML 엔지니어 추천 개선

1. **피드백 루프 구축**
   ```
   사용자 → "맞아요/아니에요" 클릭
        ↓
   Supabase에 저장
        ↓
   주간 자동 재학습
   ```

2. **실시간 모니터링**
   - 예측 신뢰도 < 70% 경고
   - 카테고리별 분포 추적
   - 오분류 케이스 로깅

3. **다양한 입력 테스트**
   ```python
   # 추가 테스트 필요한 케이스
   edge_cases = [
       "3번 틀림",  # 너무 짧음
       "몰라요 그냥",  # 정보 없음
       "ㅋㅋ 망함",  # 비표준 표현
       "multiplication error",  # 영어
   ]
   ```

### ❌ 출시 전 해결 필수

| 항목 | 상태 | 블로커? |
|------|------|---------|
| ML 모델 | ✅ 99% | No |
| 카카오 권한 | ⏳ 대기 | **Yes** |
| DB 배포 | ❌ 미완 | Yes |

### 💡 PM 추천 액션

1. 카카오 권한 승인 follow-up
2. Supabase 마이그레이션 실행
3. Edge Functions 배포

**다음 주 (사용자 검증)**
1. 5명 베타 테스터 모집
2. 실제 사진으로 테스트

**출시 후 (성장)**

2. 주간 리포트 자동 발송
