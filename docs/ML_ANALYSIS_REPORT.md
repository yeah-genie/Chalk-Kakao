## 🎯 타겟 사용자 리서치 (실제 데이터)

### 열품타 사용자 실제 피드백

**✅ 좋아하는 것:**
- 랭킹 경쟁 (자극 받음)
- 시간 측정 (성취감)
- 스트릭 (습관 형성)
- 다른 사람들 보면서 동기부여

**❌ 불만:**
- 부측(부정 측정) - 타이머만 켜놓는 사람들
- 스터디 그룹 비활성화
- 지나친 친목/경쟁
- 실제 공부 내용보다 시간만 중시

### 오답노트 앱 사용자 니즈

| 니즈 | 중요도 | Chalk 현재 |
|------|--------|-----------|
| **자동 오답 정리** (시간 절약) | ⭐⭐⭐⭐⭐ | ✅ 구현됨 |
| **주기적 복습 알림** | ⭐⭐⭐⭐ | ❌ 미구현 |
| **회독 정보 기록** | ⭐⭐⭐⭐ | ❌ 미구현 |
| **틀린 이유 분석** | ⭐⭐⭐⭐⭐ | ✅ 구현됨 |
| **취약점 파악** | ⭐⭐⭐⭐⭐ | ✅ 구현됨 |
| **PDF/필기 호환** | ⭐⭐⭐ | ❌ 미구현 |

## 🧪 ML 모델 실사용 테스트

### 1. 오류 패턴 분류 (Level 1-2)

**테스트:**
```
입력: "이항할 때 부호를 안 바꿨어요"
결과: 음수_처리 (38% 신뢰도)
```

**문제점:**
- ❌ **신뢰도가 너무 낮음** (38%)
- ❌ "이항_규칙"이 아닌 "음수_처리"로 분류됨
- ❌ 키워드 기반이라 한국어 자연어에 약함

**개선 필요:**
- 한국어 오류 설명 데이터셋 구축
- 또는 Sentence-BERT로 의미 기반 매칭

### 2. 다음 오류 예측 (Level 3)

**테스트:**
```
히스토리: [Sign Error, Sign Error, Forgot to Simplify]
예측: Sign Error (19%), Misread Problem (18%)
```

**문제점:**
- ❌ **확률이 너무 낮음** (19%)
- ❌ 실용적 가치 부족 ("19% 확률로 틀릴 것" = 별로 도움 안 됨)
- ❌ 데이터 부족 (500개로는 패턴 파악 어려움)

**개선 필요:**
- 최소 10,000+ 사용자 히스토리 필요
- 또는 "자주 반복되는 오류" 방식으로 전환

### 3. 그룹 분석

**테스트 결과:**
```
그룹 공통 취약점: Sign Error
같이 틀린 문제: 3명 모두 Sign Error
```

**평가:**
- ✅ 기능적으로 잘 동작함
- ⚠️ 하지만... **학생들이 이걸 원할까?**

### 4. 게이미피케이션

**테스트 결과:**
```
🏆 랭킹: 예진 90%, 민수 80%, 지훈 70%
🌍 전국 비교: 상위 20%
🎖️ 배지: 스트릭 마스터
```

**평가:**
- ✅ 열품타와 유사한 UX
- ⚠️ 하지만 **차별화 포인트가 없음**

## 🤔 타겟 사용자 관점 솔직한 평가

### "고3 수험생 민지"의 관점

> "음... 기능은 좋은데, 굳이 이걸 쓸 이유가?"

**열품타 vs Chalk:**

| 기능 | 열품타 | Chalk | 승자 |
|------|--------|-------|------|
| 시간 측정 | ✅ | ❌ | 열품타 |
| 랭킹 | ✅ | ✅ | 비김 |
| 스트릭 | ✅ | ✅ | 비김 |
| 스터디 그룹 | ✅ | ✅ | 비김 |
| **오답 분석** | ❌ | ✅ | **Chalk** |
| **취약점 예측** | ❌ | ✅ | **Chalk** |
| **복습 추천** | ❌ | ✅ | **Chalk** |

**결론:**
> Chalk만의 강점은 "오답 분석/예측/추천"인데,
> 현재 구현은 **정확도가 낮아서 신뢰 못함**

### "중2 스터디 그룹 '수학천재들'"의 관점

> "친구들이랑 랭킹? 열품타에서도 되는디..."
> "같이 틀린 문제? 그게 뭐 어쨌는데?"

**핵심 문제:**
- "같이 틀린 문제" 기능의 **가치 전달이 부족**
- 왜 같이 틀린 문제를 알아야 하는지 설득력 없음

---

## 📌 솔직한 판정: 실제로 쓸까?

### 현재 상태: ❌ 안 쓸 것 같음

**이유:**
1. 열품타 이미 있음 (시장 포화)
2. 차별화 기능(오답 분석)의 정확도 낮음
3. 카톡 그룹방에서 굳이?
4. "왜 이걸 써야 하지?" 답변 어려움

### 쓰게 만들려면?

**Must Have (필수):**
1. 오류 분류 정확도 80%+ (현재 30~50%)
2. 예측 확률 의미있는 수준 (60%+)
3. "아 맞아!" 느낌 주는 분석

**Killer Feature 후보:**
1. **"시험 전 복습 리스트"** - 개인 맞춤 5문제
2. **"오늘의 약점 퀴즈"** - 틀린 유형 3문제 푸시
3. **"친구랑 1:1 대결"** - 같은 유형 문제 대결

## 🚀 ML 모델 개선 로드맵

### Phase 1: 정확도 올리기 (가장 급함!)

**문제:** 한국어 자연어 이해 부족

**해결책:**
```
현재: 키워드 매칭 (TF-IDF)
목표: 의미 기반 매칭 (Sentence-BERT)

구현:
1. KoSentence-BERT 사용
2. 한국어 수학 오류 설명 데이터 수집
3. Fine-tuning
```

**예상 효과:** 정확도 30% → 75%+

### Phase 2: 실시간 학습

**문제:** 고정된 모델, 개선 안 됨

**해결책:**
```
사용자가 "맞아요/아니에요" 클릭
    ↓
Supabase에 피드백 저장
    ↓
매주 모델 재학습
    ↓
점점 똑똑해짐
```

### Phase 3: 개인화 추천

**문제:** 모든 사용자에게 같은 추천

**해결책:**
```
현재: "이항 규칙 복습하세요"
목표: "너는 음수 이항에서 특히 약해. 이 영상 봐봐"
```

### Phase 4: 문제 자체 추천

**문제:** EBS 링크만 제공

**해결책:**
```
모두매쓰 API 연동
→ AI가 실시간 문제 생성
→ "너 맞춤 연습문제 3개"
```
